# Release Plan

## v0.1.0: 프로젝트 기반 구축

- 포함 기능: Spring Boot, Gradle, Docker Compose, PostgreSQL, Redis, Swagger, 공통 응답/예외.
- 제외 기능: 인증, 캠퍼스, 도메인 기능.
- 완료 기준: 로컬/Docker 실행, CI 통과.
- 테스트 범위: 애플리케이션 context load, health check.
- Docker 확인 체크리스트: PostgreSQL/Redis 컨테이너 기동, app 컨테이너 실행.
- Secret 노출 확인 체크리스트: `.env`, DB password, JWT secret 미커밋.
- 배포 전 체크리스트: README 실행 방법 작성.
- 회고 질문: 초기 구조가 과하지 않았는가?

## v0.2.0: 인증/캠퍼스 MVP

- 포함 기능: 회원가입, 로그인, Refresh Token, 로그아웃, 캠퍼스 생성/가입/멤버 권한.
- 제외 기능: 경건생활, 청구, 투표.
- 완료 기준: 인증/캠퍼스 핵심 시나리오 통과.
- 테스트 범위: 인증 성공/실패, 토큰 재발급, 캠퍼스 가입 중복.
- Docker 확인 체크리스트: Redis token 저장 확인.
- Secret 노출 확인 체크리스트: JWT secret 미노출.
- 배포 전 체크리스트: Swagger 확인.
- 회고 질문: 인증 코드가 user 도메인에 잘 닫혀 있는가?

## v0.3.0: 경건생활/벌금 MVP

- 포함 기능: 경건생활 제출, 일별 체크, 주간 집계, 벌금 규칙, 벌금 계산, 청구 생성.
- 제외 기능: 배치 자동 생성, 알림.
- 완료 기준: 경건생활 기반 청구 생성 테스트 통과.
- 테스트 범위: 제출 중복, 주차 기준, 벌금 계산.
- Docker 확인 체크리스트: PostgreSQL 데이터 저장 확인.
- Secret 노출 확인 체크리스트: 환경변수 노출 없음.
- 배포 전 체크리스트: 벌금 규칙 문서화.
- 회고 질문: Domain Service 분리가 적절했는가?

## v0.4.0: 투표/커피비 MVP

- 포함 기능: 투표 템플릿, 투표 생성, 응답, 댓글, 커피 투표, 옵션 가격 기반 청구 생성.
- 제외 기능: 자동 알림.
- 완료 기준: Poll-Billing 연동 테스트 통과.
- 테스트 범위: 중복 응답, 마감 응답 방지, 커피비 청구.
- Docker 확인 체크리스트: 투표/청구 데이터 저장 확인.
- Secret 노출 확인 체크리스트: 없음.
- 배포 전 체크리스트: API 응답 예시 정리.
- 회고 질문: Poll과 Billing 결합도가 과하지 않은가?

## v0.5.0: 알림/자동화 MVP

- 포함 기능: FCM 토큰, 알림 발송, 미제출/미응답/미납 알림, Redis 중복 방지, Redis Lock.
- 제외 기능: 고급 알림 분석.
- 완료 기준: 중복 발송 방지와 실패 토큰 처리 테스트 통과.
- 테스트 범위: 단건/캠퍼스 알림, 실패 토큰, Redis TTL.
- Docker 확인 체크리스트: Redis key 생성/만료 확인.
- Secret 노출 확인 체크리스트: Firebase key 미커밋.
- 배포 전 체크리스트: FCM 설정 문서 작성.
- 회고 질문: Redis 목적이 명확했는가?

## v1.0.0: 운영 가능한 MVP

- 포함 기능: 관리자 조회, 캠퍼스별 집계, 납부 현황, Docker/CI/문서/포트폴리오.
- 제외 기능: MSA, 고급 통계, 앱 푸시 개인화.
- 완료 기준: 핵심 사용자 플로우와 운영자 플로우 검증 완료.
- 테스트 범위: 인증, 캠퍼스, 경건생활, 청구, 투표, 알림, 관리자 API.
- Docker 확인 체크리스트: 전체 compose 환경 실행.
- Secret 노출 확인 체크리스트: GitHub diff와 `.gitignore` 확인.
- 배포 전 체크리스트: 릴리즈 노트, README, 회고 작성.
- 회고 질문: 포트폴리오 증거가 충분히 남았는가?
