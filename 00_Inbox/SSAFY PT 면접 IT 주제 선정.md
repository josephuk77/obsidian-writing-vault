## 주제 선정 기준

SSAFY PT 면접용 IT 주제는 단순히 최신 유행어를 고르는 것보다, 개발자 관점에서 설명할 수 있는지가 중요하다.

좋은 주제의 조건:

- 개념을 짧게 정의할 수 있어야 한다.
- 왜 등장했는지 배경을 설명할 수 있어야 한다.
- 장점과 단점이 명확해야 한다.
- 단점에 대해 기술적 해결방안을 제시할 수 있어야 한다.
- 활용사례와 최근 뉴스로 현실성을 붙일 수 있어야 한다.


## 1. 클라우드와 데이터센터

### 개념

클라우드는 서버, 저장소, 네트워크, 데이터베이스 같은 IT 자원을 직접 구축하지 않고 인터넷을 통해 필요한 만큼 사용하는 방식이다.

데이터센터는 이런 클라우드와 AI 서비스를 실제로 운영하는 물리적 인프라다. 최근에는 AI 모델 학습과 추론을 위해 GPU 데이터센터의 중요성이 커지고 있다.

### 등장 배경

- 서비스 사용량이 빠르게 변하면서 서버를 직접 고정적으로 운영하기 어려워졌다.
- 트래픽 증가에 맞춰 서버를 빠르게 늘리고 줄이는 구조가 필요해졌다.
- AI 서비스가 늘면서 GPU와 대규모 데이터센터 수요가 커졌다.
- 기업은 초기 인프라 비용을 줄이고 빠르게 서비스를 출시하고 싶어 한다.

### 장점

- 서버를 빠르게 배포할 수 있다.
- 사용량에 따라 자원을 늘리거나 줄일 수 있다.
- 장애 대응과 백업 구성을 자동화하기 쉽다.
- 글로벌 서비스 확장이 쉽다.
- AI, 데이터 분석, DevOps 도구와 연결하기 좋다.

### 단점/리스크

- 특정 클라우드 사업자에 종속될 수 있다.
- 설정 실수로 보안 사고가 발생할 수 있다.
- 트래픽이 많아지면 비용이 급증할 수 있다.
- 데이터센터 전력 사용량이 커진다.
- 클라우드 장애가 발생하면 서비스 전체에 영향이 갈 수 있다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 서버 장애 | 서버 이중화, 멀티 AZ, 헬스체크, 자동 복구 |
| 트래픽 폭증 | 로드밸런싱, 오토스케일링, 캐싱, CDN |
| 비용 증가 | 리소스 모니터링, 예약 인스턴스, 서버리스, 사용량 알림 |
| 보안 설정 실수 | IAM 최소 권한, 보안 그룹 점검, IaC 코드 리뷰, CSPM |
| 클라우드 종속 | 컨테이너, Kubernetes, 멀티 클라우드/하이브리드 전략 |
| 데이터 손실 | 백업, 스냅샷, 재해복구, 다른 리전에 복제 |

### 활용사례

- 쇼핑몰 트래픽 폭증 대응
- 게임 서버 오토스케일링
- AI 모델 학습을 위한 GPU 클러스터
- 글로벌 서비스를 위한 CDN
- 장애 대응을 위한 멀티 리전 구성

### 뉴스 포인트

정부의 대규모 GPU 공급 사업에 네이버클라우드, 삼성SDS 등 주요 기업이 참여했다는 보도는 AI 시대에 클라우드와 GPU 데이터센터가 핵심 인프라가 되고 있음을 보여준다.

- 참고: [2조 GPU 사업 공모에 삼성·네이버·KT·쿠팡·엘리스 참여](https://www.fnnews.com/news/202604131835021791)

### PT 결론 문장

> 클라우드는 단순히 서버를 빌리는 기술이 아니라, 서비스의 확장성, 안정성, 장애 대응을 설계하는 기반입니다. 개발자 관점에서는 로드밸런싱, 오토스케일링, 서버 이중화, 모니터링, 백업 같은 구조를 함께 설계해야 안정적인 서비스를 만들 수 있습니다.

## 2. 사이버보안과 제로 트러스트

### 개념

제로 트러스트는 내부 네트워크라고 해서 무조건 신뢰하지 않고, 모든 접근을 계속 검증하는 보안 모델이다.

기존에는 회사 내부망에 들어오면 비교적 신뢰하는 구조가 많았다. 하지만 클라우드, 원격근무, SaaS, IoT가 늘면서 내부와 외부의 경계가 흐려졌다. 그래서 "아무도 기본적으로 믿지 않는다"는 접근이 중요해졌다.

### 등장 배경

- 클라우드와 원격근무로 접속 위치가 다양해졌다.
- 계정 탈취와 랜섬웨어 공격이 늘어났다.
- IoT, AI, API처럼 보호해야 할 대상이 많아졌다.
- 내부망 침투 이후 lateral movement가 큰 피해로 이어질 수 있다.

### 장점

- 계정 탈취 피해를 줄일 수 있다.
- 사용자, 기기, 위치, 권한을 계속 검증할 수 있다.
- 내부망 침투 이후 확산을 줄일 수 있다.
- 클라우드와 원격근무 환경에 잘 맞는다.

### 단점/리스크

- 초기 구축 비용과 운영 복잡도가 크다.
- 인증 절차가 많아지면 사용자 경험이 나빠질 수 있다.
- 기존 시스템과 연동이 어려울 수 있다.
- 정책을 잘못 설정하면 정상 사용자가 차단될 수 있다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 계정 탈취 | MFA, 패스키, 이상 로그인 탐지, 세션 만료 |
| 과도한 권한 | RBAC/ABAC, 최소 권한, 권한 주기적 검토 |
| 내부망 확산 | 마이크로 세그멘테이션, 네트워크 분리, 접근 제어 |
| API 공격 | API Gateway, Rate Limit, JWT 검증, WAF |
| 보안 로그 누락 | SIEM, 감사 로그, 알림 규칙, 이상 행위 탐지 |
| 사용자 불편 | 위험 기반 인증, SSO, 조건부 접근 |

### 활용사례

- 사내 시스템 로그인 보안
- 클라우드 관리자 권한 보호
- IoT 기기 접근 제어
- API 호출 제한
- 금융/공공 서비스 접근 제어

### 뉴스 포인트

KISA가 AI 보안과 제로 트러스트 등 차세대 보안기술 지원사업을 추진한다는 보도는 보안이 단순 운영 문제가 아니라 핵심 IT 기술 주제로 커지고 있음을 보여준다.

- 참고: [AI 보안·제로트러스트 키운다…KISA, 18개 과제 선정](https://www.etnews.com/20260508000063)

### PT 결론 문장

> 제로 트러스트는 내부망도 무조건 신뢰하지 않고 모든 접근을 검증하는 보안 모델입니다. 개발자 관점에서는 MFA, 최소 권한, API Gateway, 감사 로그, 네트워크 분리 같은 기술을 통해 보안 리스크를 줄일 수 있습니다.

## 3. 엣지 컴퓨팅과 온디바이스 AI

### 개념

엣지 컴퓨팅은 데이터를 멀리 있는 클라우드로 보내기 전에 데이터가 발생하는 현장 가까이에서 처리하는 방식이다.

온디바이스 AI는 스마트폰, 카메라, 자동차, 웨어러블 기기 같은 기기 내부에서 AI 추론을 수행하는 방식이다.

### 등장 배경

- IoT 기기가 늘면서 클라우드로 보내야 할 데이터가 많아졌다.
- 자율주행, 스마트팩토리, 헬스케어처럼 지연 시간이 중요한 서비스가 늘었다.
- 개인정보를 외부 서버로 보내지 않고 처리해야 하는 요구가 커졌다.
- AI 반도체와 모바일 NPU 성능이 좋아졌다.

### 장점

- 응답 속도가 빠르다.
- 네트워크가 불안정해도 일부 기능을 유지할 수 있다.
- 개인정보 보호에 유리하다.
- 클라우드 전송 비용을 줄일 수 있다.
- 실시간 제어가 필요한 서비스에 적합하다.

### 단점/리스크

- 기기 성능과 배터리 한계가 있다.
- 모델 업데이트가 어렵다.
- 기기별 환경이 달라 품질 관리가 어렵다.
- 현장 기기가 해킹되면 위험할 수 있다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 기기 성능 부족 | 모델 경량화, 양자화, 캐싱, NPU 활용 |
| 모델 업데이트 문제 | OTA 업데이트, 버전 관리, 단계적 배포, 롤백 |
| 네트워크 장애 | 로컬 큐, 재전송, 오프라인 모드 |
| 보안 문제 | Secure Boot, 기기 인증, 암호화, 원격 무결성 검증 |
| 품질 차이 | 기기별 테스트, 원격 모니터링, A/B 테스트 |

### 활용사례

- 스마트폰 음성 인식
- CCTV 이상 행동 감지
- 자율주행 차량의 실시간 판단
- 스마트팩토리 설비 이상 감지
- 웨어러블 건강 데이터 분석

### 뉴스 포인트

2026 월드IT쇼에서 피지컬 AI, 디지털 트윈, 로봇 트레이닝 플랫폼이 소개된 흐름은 AI가 클라우드 안에서만 머무르지 않고 현실 세계의 기기와 현장으로 확장되고 있음을 보여준다.

- 참고: [SKT, 월드IT쇼에서 인프라부터 서비스까지 풀스택 AI 선보인다](https://news.sktelecom.com/223989)

### PT 결론 문장

> 엣지 컴퓨팅과 온디바이스 AI는 데이터를 현장 가까이에서 처리해 지연 시간, 개인정보, 네트워크 의존성 문제를 줄이는 기술입니다. 개발자는 모델 경량화, OTA 업데이트, 로컬 큐, 기기 인증 같은 구조까지 함께 고려해야 합니다.

## 4. 디지털 트윈과 스마트팩토리

### 개념

디지털 트윈은 현실의 설비, 공장, 도시, 차량 같은 대상을 디지털 공간에 복제해 모니터링하고 시뮬레이션하는 기술이다.

스마트팩토리는 공장 설비와 생산 데이터를 연결해 생산성, 품질, 유지보수를 개선하는 공장 운영 방식이다.

### 등장 배경

- 제조 현장의 설비와 공정이 복잡해졌다.
- 고장이 난 뒤 수리하는 방식은 비용이 크다.
- 생산 데이터를 기반으로 품질과 효율을 개선해야 한다.
- AI와 IoT가 발전하면서 현실 공정을 디지털로 분석할 수 있게 되었다.

### 장점

- 설비 상태를 실시간으로 확인할 수 있다.
- 고장을 미리 예측할 수 있다.
- 공정 변경 전 시뮬레이션을 해볼 수 있다.
- 불량률과 생산 비용을 줄일 수 있다.
- 작업자 안전을 높일 수 있다.

### 단점/리스크

- 센서와 데이터 품질이 낮으면 분석 결과가 부정확하다.
- 초기 구축 비용이 크다.
- 기존 설비와 시스템 연동이 어렵다.
- 공장 데이터 유출 위험이 있다.
- 시스템 장애가 생산 중단으로 이어질 수 있다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 데이터 품질 문제 | 센서 캘리브레이션, 이상치 탐지, 데이터 검증 파이프라인 |
| 시스템 장애 | 서버 이중화, 엣지 처리, 로컬 제어, 장애 감지 |
| 기존 설비 연동 | 표준 프로토콜, 게이트웨이, 메시지 브로커 |
| 보안 문제 | 망 분리, 접근 제어, 암호화, 감사 로그 |
| 예측 모델 오류 | 모델 모니터링, 재학습, 휴먼 검토, 롤백 |

### 활용사례

- 공장 설비 예지보전
- 생산 라인 병목 분석
- 불량품 자동 검사
- 물류 동선 최적화
- 도시 인프라 시뮬레이션

### 뉴스 포인트

스마트팩토리와 AI 기반 자율 제조공장 모델 관련 보도는 제조 분야에서 AI와 IoT, 디지털 트윈이 실제 산업 문제 해결에 쓰이고 있음을 보여준다.

- 참고: [AI 기반 자율 제조공장 모델 공개](https://www.mt.co.kr/future/2026/03/26/2026032615230022508)

### PT 결론 문장

> 디지털 트윈은 현실의 공장이나 설비를 디지털 공간에 복제해 모니터링과 시뮬레이션을 가능하게 하는 기술입니다. 개발자 관점에서는 센서 데이터 품질, 메시지 처리, 장애 대응, 보안, 모델 모니터링까지 함께 설계해야 합니다.

## 5. 스마트 모빌리티

### 개념

스마트 모빌리티는 차량, 대중교통, 공유 킥보드, 자율주행, 교통 인프라를 데이터와 네트워크로 연결해 이동을 더 효율적이고 안전하게 만드는 기술이다.

### 등장 배경

- 도시 교통 혼잡이 커졌다.
- 위치 기반 서비스와 모바일 결제가 보편화되었다.
- 자율주행, 전기차, 공유 이동수단이 확산되고 있다.
- 교통 데이터를 기반으로 효율적인 이동을 설계할 필요가 커졌다.

### 장점

- 이동 경로를 최적화할 수 있다.
- 교통 혼잡을 줄일 수 있다.
- 차량과 인프라 데이터를 활용해 안전을 높일 수 있다.
- 공유 이동수단을 효율적으로 운영할 수 있다.
- 장애인, 고령자 등 교통 약자의 이동 편의성을 높일 수 있다.

### 단점/리스크

- 위치 개인정보 유출 위험이 있다.
- 인증 서버 장애 시 서비스 이용이 막힐 수 있다.
- 실시간 교통 데이터가 틀리면 잘못된 안내가 나올 수 있다.
- 결제 시스템 장애가 이동 불편으로 이어질 수 있다.
- 자율주행이나 자동 제어는 안전 문제가 크다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 위치정보 유출 | 위치 데이터 최소 수집, 익명화, 암호화, 보관 기간 제한 |
| 인증 장애 | 서버 이중화, 대체 인증, 로컬 임시 토큰, 장애 모드 |
| 트래픽 폭증 | 로드밸런싱, 캐싱, 오토스케일링 |
| 결제 장애 | 결제 재시도 큐, 장애 알림, 대체 결제 수단 |
| 실시간성 문제 | 엣지 컴퓨팅, 메시지 큐, 스트리밍 처리 |
| 안전 문제 | 이중 센서, Fail-safe, 수동 제어, 시뮬레이션 테스트 |

### 활용사례

- 버스 도착 예측
- 공유 킥보드 위치 관리
- 자율주행 차량
- 스마트 주차
- 교통 신호 최적화

### PT 결론 문장

> 스마트 모빌리티는 이동 데이터를 활용해 교통 효율과 안전을 높이는 기술입니다. 다만 위치정보, 인증 장애, 결제 장애, 실시간성 문제가 크기 때문에 서버 이중화, 로드밸런싱, 엣지 컴퓨팅, 대체 인증, Fail-safe 같은 기술적 설계가 필요합니다.

## 6. 빅데이터와 데이터 파이프라인

### 개념

빅데이터는 기존 방식으로 처리하기 어려울 정도로 크고 빠르게 생성되는 데이터를 수집, 저장, 처리, 분석하는 기술이다.

데이터 파이프라인은 데이터를 발생 지점에서 수집해 저장소나 분석 시스템으로 전달하는 전체 흐름이다.

### 등장 배경

- 웹, 앱, IoT, 결제, 로그 데이터가 계속 증가하고 있다.
- AI 모델 학습과 추천 시스템에 대량의 데이터가 필요하다.
- 기업은 감이 아니라 데이터 기반으로 의사결정을 하고 싶어 한다.

### 장점

- 사용자 행동을 분석할 수 있다.
- 추천과 개인화가 가능하다.
- 이상 징후를 빠르게 찾을 수 있다.
- AI 모델 학습 데이터로 활용할 수 있다.
- 비즈니스 의사결정 근거를 만들 수 있다.

### 단점/리스크

- 데이터 품질이 낮으면 분석 결과도 틀린다.
- 개인정보 유출 위험이 있다.
- 실시간 처리와 배치 처리를 함께 설계해야 한다.
- 데이터 저장 비용이 커질 수 있다.
- 데이터 사일로가 생길 수 있다.

### 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 데이터 품질 문제 | 스키마 검증, 이상치 탐지, 데이터 카탈로그 |
| 실시간 처리 | Kafka, 메시지 큐, 스트리밍 처리 |
| 개인정보 문제 | 마스킹, 익명화, 접근 권한, 암호화 |
| 저장 비용 | 데이터 수명주기 관리, 압축, 콜드 스토리지 |
| 장애 대응 | 재처리 가능한 파이프라인, 체크포인트, 모니터링 |

### PT 결론 문장

> 빅데이터는 AI와 서비스 개선의 기반이지만, 단순히 많이 모으는 것이 아니라 품질, 보안, 저장 비용, 실시간 처리 구조를 함께 설계해야 의미 있는 데이터가 됩니다.
