# 📘 Week 01 — 코딩테스트 준비하기 (시간복잡도 · 디버깅)

# 시간 복잡도 표기법 알아보기
## 시간 복잡도 유형
- 빅-오메가(Ω(n)): 최선일 때(best case)의 연산 횟수를 나태낸 표기법
- 빅-세타(θ(n)): 보통일 때(average case)의 연산 횟수를 나타낸 표기법
- 빅-오(O(n)): 최악일 때(worst case)의 연산 횟수를 나타낸 표기법

### 시간복잡도 예제 코드
```cpp
#include <iostream>
#include <cstdlib>
#include <ctime>
using namespace std;

int main()
{
	int findNumber;
    srand(time(NULL));
    findNUmber = rand() % 100;	//0~99 사이의 값 무작위 선택
    
    for (int i = 0; i < 100; i++) {
    	if (i == findNumber) {
        	cout << i;
            break;
        }
    }
}
```
> 코딩 테스트에서는 빅-오 표기법(O(n))을 기준으로 수행 시간을 계산하는 것이 좋다.
응시자가 작성한 프로그램으로 다양한 테스트 케이스를 수행해 모든 케이스를 통과해야만 합격으로 판단하므로 시간 복잡도를 판단할 때는 최악일 때를 염두해야 한다

# 시간 복잡도 예제
### 예제 1: 연산 횟수 = N
```cpp
#include <iostream>
using namespace std;

int main() 
{
	int N = 1000;
    int cnt = 1;
    
    for (int i = 0; i < N; i++) {
    	cout << "연산 횟수: " << cnt++ << "\n";
    }
}
```
### 예제 2: 연산 횟수 = 3N
```cpp
#include <iostream>
using namespace std;

int main() 
{
	int N = 1000;
    int cnt = 1;
    
    for (int i = 0; i < N; i++) {
    	cout << "연산 횟수: " << cnt++ << "\n";
    }
    for (int i = 0; i < N; i++) {
    	cout << "연산 횟수: " << cnt++ << "\n";
    }
    for (int i = 0; i < N; i++) {
    	cout << "연산 횟수: " << cnt++ << "\n";
    }
}
```
### 예제 3: 연산 횟수 = N²
```cpp
#include <iostream>
using namespace std;

int main() 
{
	int N = 1000;
    int cnt = 1;
    
    for (int i = 0; i < N; i++) {
    	for (int j = 0; j < N; j++) {
    		cout << "연산 횟수: " << cnt++ << "\n";
        }
    }
    
}
```
# 디버깅은 왜 중요할까

## 디버깅의 중요성

디버깅(Debugging)은 프로그램을 한 줄씩 실행하면서 변수의 값과 코드 흐름을 확인해 오류를 찾는 과정이다.

문법 오류는 컴파일러가 알려주지만 논리 오류는 직접 찾아야 한다.  
특히 코딩 테스트에서는 다양한 테스트 케이스를 통과해야 하기 때문에 디버깅을 통해 문제의 원인을 빠르게 찾는 능력이 중요하다.

---

## 디버깅 하는 법

개발 툴에서 제공하는 디버거(Debugger)를 사용하면 프로그램을 한 줄씩 실행하면서 변수의 변화를 확인할 수 있다.

주로 다음과 같은 방법을 사용한다.

- 브레이크 포인트(Break Point) 설정하기
- 한 줄씩 실행하며 코드 흐름 확인하기
- 변수 값이 예상대로 변하는지 확인하기

이를 통해 반복문이나 조건문에서 발생하는 논리 오류를 쉽게 찾을 수 있다.

---

## 디버깅 활용 사례 살펴보기

디버거를 사용하면 특정 위치에서 프로그램을 멈추고 현재 변수 상태를 확인할 수 있다.

반복문이 예상보다 많이 실행되거나 조건문이 동작하지 않는 경우 변수 값을 직접 확인하면서 원인을 찾을 수 있다.

---

# 코딩 테스트를 진행하며 실수하기 쉬운 오류 찾아보기

## 오류 1 변수 초기화 오류 찾아보기

초기화하지 않은 변수는 예상하지 못한 값을 가질 수 있다.

### 잘못된 예
```cpp
int sum;
sum += 10;
```
### 올바른 예
```cpp
int sum = 0;
```
---

## 오류 2 반복문에서 인덱스 범위 지정 오류 찾아보기

배열의 범위를 벗어나면 런타임 에러가 발생할 수 있다.

### 잘못된 예
```cpp
for(int i = 0; i <= N; i++)
```
### 올바른 예
```cpp
for(int i = 0; i < N; i++)
```
---

## 오류 3 잘못된 변수 사용 오류 찾아보기

비슷한 변수 이름을 사용할 때 자주 발생한다.

### 잘못된 예
```cpp
for(int i = 0; i < N; i++) {
    sum += j;
}
```
### 올바른 예
```cpp
for(int i = 0; i < N; i++) {
    sum += i;
}
```
