# 📘 TIL - 백준 1032번: 명령 프롬프트

🔍 오늘 배운 것
📌 문제 개요
윈도우 명령 프롬프트(cmd)에서 dir 명령어는 특정 패턴에 맞는 파일만 출력합니다.

예를 들어 dir a?c.txt는 abc.txt, acc.txt 등을 찾습니다.

주어진 N개의 파일 이름에서, 해당 결과가 나올 수 있는 패턴 문자열을 찾는 문제입니다.

✅ 문제 조건 정리
파일 개수 N ≤ 50

파일 길이 ≤ 50

모두 동일한 길이의 파일 이름

**알파벳 소문자와 점(.)**으로 구성

가능한 한 ?를 최소로 사용해야 함

💡 문제 해결 전략
1. 비교 기준 설정
첫 번째 파일 이름을 기준으로 잡고, 다른 파일들과 각 문자의 위치를 비교

2. 문자열 비교 로직
같은 위치에 있는 문자가 모두 동일 → 해당 문자 사용

하나라도 다르면 → ?로 대체

3. 예외 처리
입력 파일이 1개일 경우 → 그 파일 이름 그대로 출력

🧠 배운 점
문자열 문제는 위치 기준 비교로 해결하는 방식이 유용하다.

비교 시 기준 문자열을 하나 정해두면 비교 횟수를 줄일 수 있다.

Scanner보다 BufferedReader가 속도 면에서 유리하다 (입력이 많을 때 특히 중요).

자바에서 문자 비교는 charAt(i)를 사용하는 방식이 직관적이고 효율적이다.

문자열 결과를 만들 땐 StringBuilder를 사용하는 것이 좋다.

🧑‍💻 사용한 코드 요약
```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
int n = Integer.parseInt(br.readLine());
String[] files = new String[n];

for (int i = 0; i < n; i++) {
    files[i] = br.readLine();
}

if (n == 1) {
    System.out.println(files[0]);
    return;
}

StringBuilder sb = new StringBuilder();

for (int i = 0; i < files[0].length(); i++) {
    char ch = files[0].charAt(i);
    boolean same = true;

    for (int j = 1; j < n; j++) {
        if (files[j].charAt(i) != ch) {
            same = false;
            break;
        }
    }

    sb.append(same ? ch : '?');
}

System.out.println(sb.toString());
```
⏱️ 복잡도 분석
시간 복잡도: O(N × L)
(N = 파일 수, L = 문자열 길이)

공간 복잡도: O(L)
(출력 문자열 길이만큼 저장)

🔗 느낀 점
처음에는 단순히 문자열 하나씩 비교하는 게 비효율적일 것 같았지만, 문자열 길이가 고정되어 있어 충분히 가능한 문제였다.

조건을 잘 활용하면 로직이 훨씬 단순해질 수 있다는 점을 다시 느꼈다.

앞으로 문자열 관련 문제는 "기준값 설정 + 위치별 비교" 패턴을 적극적으로 활용해야겠다.
