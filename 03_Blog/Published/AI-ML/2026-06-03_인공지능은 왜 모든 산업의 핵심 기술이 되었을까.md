# 인공지능은 왜 모든 산업의 핵심 기술이 되었을까

## 개념 한 줄 정리

인공지능을 말할 때는 하나의 기술만 떠올리기보다, 아래 개념들이 어떻게 이어지는지 먼저 잡는 것이 좋다.

| 개념 | 간단한 설명 |
| --- | --- |
| 인공지능 | 컴퓨터가 사람처럼 학습, 판단, 문제 해결을 하도록 만드는 넓은 기술 |
| 머신러닝 | 사람이 규칙을 전부 코딩하지 않고, 데이터에서 패턴을 학습하게 하는 방식 |
| 딥러닝 | 인공신경망을 여러 층으로 쌓아 이미지, 음성, 자연어 같은 복잡한 데이터를 학습하는 방식 |
| 생성형 AI | 텍스트, 이미지, 코드, 영상처럼 새로운 결과물을 만들어내는 AI |
| 프롬프트 엔지니어링 | AI에게 원하는 답을 얻기 위해 질문, 역할, 조건, 출력 형식을 설계하는 방법 |
| RAG | 외부 문서나 지식베이스를 검색한 뒤, 그 내용을 근거로 답변을 생성하는 방식 |
| AI 에이전트 | 목표를 받고 계획을 세우며 도구를 사용해 작업을 수행하는 AI |
| 엣지 AI/온디바이스 AI | 클라우드가 아니라 스마트폰, 카메라, 공장 장비 같은 기기 가까이에서 AI를 실행하는 방식 |
| AIoT | IoT가 수집한 현실 데이터를 AI가 분석해 예측, 판단, 자동 제어를 수행하는 구조 |

## 글을 쓰는 이유

PT 면접에서 인공지능을 주제로 이야기하려면 "요즘 AI가 중요합니다" 정도로는 부족하다.

면접관이 듣고 싶은 것은 기술 이름을 아는지가 아니라, 그 기술이 왜 필요한지, 어디에 쓰이는지, 어떤 한계가 있는지, 그리고 다른 기술과 어떻게 연결되는지를 이해하고 있는지다.

특히 이번 주제는 인공지능과 IoT다. IoT는 현실 세계의 데이터를 수집하고, 인공지능은 그 데이터를 바탕으로 판단과 예측을 수행한다. 센서가 눈과 귀라면, AI는 그 데이터를 해석하는 두뇌에 가깝다.

## 먼저 결론부터

인공지능은 단순히 사람의 일을 대신하는 기술이 아니다.

더 정확히 말하면 인공지능은 데이터를 기반으로 판단, 예측, 생성, 자동화를 수행하는 기술이다. 과거의 소프트웨어가 사람이 정한 규칙대로 동작했다면, 인공지능은 데이터에서 패턴을 학습하고 새로운 상황에 대한 답을 만들어낸다.

그리고 IoT와 결합하면 인공지능의 가치는 더 커진다. IoT 기기가 현장에서 데이터를 계속 수집하고, AI가 그 데이터를 분석하면 설비 고장을 미리 예측하거나, 스마트홈 환경을 자동으로 조절하거나, 교통 흐름을 최적화하는 일이 가능해진다.

하지만 인공지능은 만능이 아니다. 데이터 품질이 낮으면 결과도 흔들리고, 개인정보나 편향 문제도 생길 수 있다. 그래서 인공지능을 잘 활용한다는 것은 모델을 도입하는 것만이 아니라, 데이터 관리와 검증 체계를 함께 설계하는 것이다.

## 개념

### 인공지능, 머신러닝, 딥러닝

인공지능은 가장 넓은 개념이다. 사람이 하는 것처럼 학습하고, 판단하고, 문제를 해결하는 시스템을 만들려는 기술 전체를 말한다.

머신러닝은 인공지능을 구현하는 방법 중 하나다. 사람이 모든 규칙을 직접 코딩하지 않고, 데이터를 통해 컴퓨터가 패턴을 학습하게 만든다. 예를 들어 이메일이 스팸인지 아닌지 판단하는 모델은 수많은 이메일 데이터를 보고 스팸의 특징을 학습할 수 있다.

딥러닝은 머신러닝 안에 포함되는 기술이다. 여러 층으로 구성된 인공신경망을 사용해 더 복잡한 패턴을 학습한다. 이미지 인식, 음성 인식, 자연어 처리처럼 데이터가 복잡하고 양이 많을 때 강점을 가진다.

```text
인공지능
-> 머신러닝
-> 딥러닝
```

면접에서 사용할 문장:

> 인공지능은 가장 넓은 개념이고, 머신러닝은 데이터를 통해 학습하는 방식이며, 딥러닝은 인공신경망을 여러 층으로 쌓아 복잡한 패턴을 학습하는 머신러닝의 한 분야입니다.

### 생성형 AI

생성형 AI는 텍스트, 이미지, 코드, 영상처럼 새로운 결과물을 만들어내는 AI다.

기존 AI가 주로 분류, 예측, 추천에 강했다면 생성형 AI는 결과물을 직접 만든다. 예를 들어 고객 문의를 분류하는 것에서 끝나는 것이 아니라, 답변 초안을 작성하거나 보고서 요약본을 만들 수 있다.

그래서 생성형 AI는 일반 사용자도 변화를 바로 체감하기 쉽다. 문서 작성, 요약, 번역, 코드 작성, 이미지 생성처럼 일상적인 작업에 바로 연결되기 때문이다.

### RAG

RAG는 생성형 AI가 외부 문서나 지식베이스를 먼저 검색하고, 그 내용을 바탕으로 답변하게 만드는 방식이다.

생성형 AI는 학습하지 않은 최신 정보나 회사 내부 문서에 약하다. 이때 RAG를 사용하면 회사 규정, 제품 매뉴얼, 기술 문서, 개인 노트처럼 모델이 원래 알지 못하는 자료를 답변 근거로 사용할 수 있다.

```text
질문 입력
-> 관련 문서 검색
-> 검색된 문서를 AI에게 함께 제공
-> 문서를 근거로 답변 생성
```

Obsidian으로 RAG를 구성하는 것도 같은 맥락이다. 내가 정리한 노트와 프로젝트 기록을 AI가 검색할 수 있게 만들면, 단순한 메모 앱이 아니라 개인 지식 기반 AI 검색 도구처럼 활용할 수 있다.

### AI 에이전트

AI 에이전트는 단순히 질문에 답하는 AI보다 한 단계 더 실행에 가깝다.

일반 챗봇은 "무엇인가요?"라는 질문에 답한다. 반면 AI 에이전트는 "경쟁사 분석 보고서를 만들어줘" 같은 목표를 받고, 필요한 단계를 계획하고, 도구를 사용해 작업을 수행한다.

다만 AI가 도구를 사용하고 여러 단계를 수행할수록 잘못된 방향으로 갈 가능성도 커진다. 그래서 역할, 권한, 검증 절차를 정해두는 것이 중요하다. 개발 분야에서는 이런 통제 구조를 하네스라고 부를 수 있다.

### AI와 IoT

IoT는 센서, 기기, 설비, 가전제품, 차량 같은 물리적 장치가 네트워크에 연결되어 데이터를 주고받는 구조다.

IoT만 있으면 데이터를 수집할 수 있다. 하지만 데이터를 수집하는 것만으로는 충분하지 않다. 중요한 것은 그 데이터를 해석하고, 의미 있는 판단으로 바꾸는 것이다. 여기에 인공지능이 들어간다.

```text
센서가 현실 데이터를 수집한다.
-> 네트워크를 통해 데이터가 전달된다.
-> AI가 데이터를 분석한다.
-> 이상 징후를 예측하거나 자동 제어를 수행한다.
-> 사용자나 현장 시스템에 결과가 전달된다.
```

이 조합을 AIoT라고 부르기도 한다.

### 엣지 AI와 온디바이스 AI

AI와 IoT가 결합될 때 모든 데이터를 클라우드로 보내야 하는 것은 아니다.

엣지 AI 또는 온디바이스 AI는 스마트폰, 카메라, 자동차, 공장 설비, 웨어러블 기기처럼 데이터가 발생하는 곳 가까이에서 AI 추론을 수행하는 방식이다.

이 방식은 응답 속도가 빠르고, 민감한 데이터를 외부 서버로 보내지 않아도 되며, 네트워크가 불안정한 환경에서도 활용할 수 있다는 장점이 있다.

## 등장 배경

인공지능이 최근 더 중요해진 이유는 알고리즘 하나가 좋아졌기 때문만은 아니다. 데이터, 연산 자원, 서비스 수요가 동시에 커졌기 때문이다.

- 데이터가 폭발적으로 늘어났다. 스마트폰, 웹 서비스, IoT 기기, 결제 시스템, 물류 시스템에서 사용자의 행동과 현실 세계의 상태가 계속 데이터로 쌓이고 있다. AI는 이 데이터를 바탕으로 패턴을 학습한다.

- GPU와 클라우드 같은 연산 인프라가 발전했다. 예전에는 대규모 모델을 학습하고 운영하는 비용이 컸지만, 지금은 클라우드와 AI 반도체 발전으로 더 큰 모델을 더 빠르게 다룰 수 있게 되었다.

- 기업의 자동화 수요가 커졌다. 기업은 반복 업무를 줄이고, 고객 응대를 빠르게 하고, 데이터 기반 의사결정을 강화하고 싶어 한다. 생성형 AI는 문서 작성, 요약, 번역, 코드 작성처럼 실제 업무와 바로 연결되기 때문에 도입 속도가 더 빨라졌다.

- IoT와 연결된 현실 데이터가 늘어나고 있다. 공장 설비, 차량, 스마트홈, 헬스케어 기기가 계속 데이터를 만들고 있기 때문에 AI가 분석할 수 있는 대상도 넓어졌다.

정리하면 인공지능은 단순한 유행이 아니라, 데이터 증가와 연산 인프라 발전, 자동화 수요가 함께 만든 흐름이다.

## 장점

인공지능의 장점은 단순히 "편리하다"로 끝나지 않는다.

- 반복 업무를 자동화할 수 있다. 문서 요약, 고객 상담, 데이터 분류, 코드 생성 같은 작업에서 시간을 줄일 수 있다.

- 사람이 놓치기 쉬운 패턴을 찾을 수 있다. 제조 설비의 미세한 이상 징후나 금융 거래의 이상 패턴처럼 데이터가 많고 복잡한 영역에서 강점을 가진다.

- 실시간 예측과 대응이 가능하다. IoT 데이터와 결합하면 현장에서 발생하는 변화를 빠르게 감지하고 자동으로 대응할 수 있다.

- 개인화된 서비스를 만들 수 있다. 추천 시스템, 맞춤형 교육, 개인 건강관리처럼 사용자마다 다른 결과를 제공할 수 있다.

- 위험한 환경에서 사람의 판단을 보조할 수 있다. 공장 설비 점검, 의료 영상 판독, 보안 이상 탐지처럼 사람이 모든 데이터를 직접 확인하기 어려운 영역에서 도움이 된다.

## 단점

인공지능은 강력하지만 한계도 분명하다.

- 데이터 품질에 크게 의존한다. AI는 데이터로부터 학습하기 때문에 데이터가 부족하거나 편향되어 있으면 결과도 왜곡된다.

- 개인정보 문제가 생길 수 있다. IoT와 결합할수록 위치, 건강, 생활 패턴 같은 민감한 데이터가 늘어난다.

- 결과를 설명하기 어려울 수 있다. AI가 어떤 근거로 판단했는지 설명하기 어렵다면 의료, 금융, 채용처럼 중요한 영역에서는 신뢰하기 어렵다.

- 생성형 AI는 그럴듯하지만 틀린 정보를 만들 수 있다. 이를 환각이라고 부른다.

- 자동화가 늘어나면서 일부 직무 구조가 바뀔 수 있다. 단순 반복 업무는 줄어들 수 있지만, AI를 관리하고 검증하는 역량은 더 중요해진다.

## 해결방안

인공지능의 문제를 줄이려면 "주의해야 한다"는 말에서 끝나면 안 된다. 개발자 관점에서는 리스크별로 어떤 기술적 장치를 둘 수 있는지까지 설명해야 설득력이 생긴다.

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 학습 데이터 품질 문제 | 데이터 검증 파이프라인, 중복/이상치 제거, 라벨 품질 관리, 데이터 버전 관리 |
| 편향된 결과 | 데이터 분포 점검, 편향 지표 모니터링, 샘플링 보정, 모델 평가 기준 분리 |
| 개인정보 유출 | 개인정보 마스킹, 익명화, 암호화, 접근 권한 관리, 민감 정보 필터링 |
| 생성형 AI 환각 | RAG 적용, 답변 출처 표시, 검색 문서 기반 응답, 신뢰도 낮은 답변 차단 |
| 잘못된 의사결정 | 휴먼 인 더 루프, 승인 워크플로우, 고위험 작업 자동 실행 제한 |
| 설명 가능성 부족 | 로그 저장, 입력/출력 추적, 모델 판단 근거 기록, 설명 가능한 AI 기법 활용 |
| 모델 성능 저하 | 운영 지표 모니터링, 드리프트 감지, A/B 테스트, 모델 재학습 및 롤백 |
| AI 에이전트 오작동 | 도구 권한 제한, 샌드박스 실행, 작업 범위 제한, 감사 로그, 승인 기반 실행 |

예를 들어 생성형 AI가 틀린 답을 만드는 문제는 "사람이 조심해야 한다"로 끝내기보다 RAG를 적용해 내부 문서나 최신 문서를 검색하게 하고, 답변에 출처를 붙이고, 중요한 답변은 사람이 승인하도록 만들 수 있다.

AI 에이전트가 잘못된 파일을 수정하거나 외부 API를 호출하는 위험은 샌드박스, 최소 권한, 작업 범위 제한, 실행 전 승인 절차로 줄일 수 있다.

면접에서 사용할 문장:

> 인공지능은 도입 자체보다 운영과 통제가 중요합니다. 개발자 관점에서는 RAG, 로그 추적, 권한 제한, 모니터링, 휴먼 인 더 루프, 모델 롤백 같은 기술적 장치를 함께 설계해야 실무에서 신뢰할 수 있습니다.

## 활용사례

### 제조

스마트팩토리에서는 설비의 진동, 온도, 전류 데이터를 수집한다. AI는 이 데이터를 분석해 평소와 다른 패턴을 찾고, 고장이 발생하기 전에 점검이 필요하다고 알려줄 수 있다. 이것이 예지보전이다.

또한 카메라와 딥러닝을 활용해 제품 불량을 자동으로 검사할 수 있다.

### 의료

의료 분야에서는 영상 판독 보조, 질병 예측, 환자 모니터링에 AI를 활용할 수 있다.

예를 들어 웨어러블 기기가 심박수나 수면 데이터를 수집하고, AI가 이상 신호를 분석해 사용자에게 알려줄 수 있다.

### 금융

금융에서는 이상 거래 탐지, 신용 평가, 상담 자동화에 AI가 쓰인다.

거래 패턴을 분석해 평소와 다른 이상 거래를 찾거나, 고객 문의를 분류하고 답변 초안을 생성할 수 있다.

### 유통과 서비스

유통 분야에서는 상품 추천, 수요 예측, 물류 최적화에 AI를 활용한다.

고객의 구매 이력과 행동 데이터를 분석해 개인화된 추천을 제공하고, 재고 부족이나 과잉을 줄일 수 있다.

### 개발과 지식관리

개발자는 생성형 AI를 코드 작성, 테스트 코드 생성, 문서 작성, 오류 원인 분석에 활용할 수 있다.

또한 Obsidian 같은 개인 문서 저장소를 RAG로 연결하면, 내가 쌓아둔 노트를 AI가 검색해서 답변하는 개인 지식관리 도구로 활용할 수 있다.

### 스마트홈과 스마트시티

스마트홈에서는 사용자의 생활 패턴을 학습해 조명, 냉난방, 가전제품을 자동으로 조절할 수 있다.

스마트시티에서는 교통량 데이터를 분석해 신호 체계를 조정하거나, CCTV와 센서 데이터를 활용해 도시 안전을 높일 수 있다.

## 예상 질문과 답변

### 인공지능과 머신러닝의 차이는 무엇인가요?

인공지능은 사람의 지능적 행동을 컴퓨터가 수행하도록 만드는 넓은 개념입니다. 머신러닝은 그중에서도 데이터를 통해 패턴을 학습하고 예측하는 방식입니다.

### 생성형 AI는 기존 AI와 무엇이 다른가요?

기존 AI가 주로 분류, 예측, 추천에 집중했다면 생성형 AI는 텍스트, 이미지, 코드처럼 새로운 결과물을 만들어낼 수 있다는 점이 다릅니다.

### RAG는 왜 필요한가요?

생성형 AI는 최신 정보나 내부 문서에 약하고, 모르는 내용을 그럴듯하게 말할 수 있습니다. RAG는 외부 문서를 검색해서 그 내용을 근거로 답변하게 하므로 정확성과 신뢰성을 높이는 데 도움이 됩니다.

### IoT와 AI가 결합되면 어떤 장점이 있나요?

IoT는 현실 세계의 데이터를 수집하고, AI는 그 데이터를 분석합니다. 둘이 결합되면 설비 이상 감지, 스마트홈 자동 제어, 헬스케어 모니터링처럼 실시간 예측과 대응이 가능해집니다.

### AI의 잘못된 판단은 어떻게 줄일 수 있나요?

데이터 품질을 관리하고, 중요한 의사결정에는 사람의 검토를 넣어야 합니다. 기술적으로는 RAG로 근거 문서를 붙이고, 입력과 출력 로그를 남기고, 모델 성능을 모니터링하며, 문제가 생겼을 때 이전 모델로 롤백할 수 있는 구조가 필요합니다.

## 정리

인공지능은 이제 특정 산업만의 기술이 아니다. 검색, 추천, 번역, 챗봇, 자율주행, 스마트팩토리, 헬스케어처럼 거의 모든 분야에 들어가고 있다.

내가 이 주제를 PT 면접에서 말한다면 핵심은 하나다.

> 인공지능은 데이터를 기반으로 더 나은 판단과 자동화를 가능하게 하는 기술입니다. 특히 IoT와 결합하면 현실 세계에서 발생하는 데이터를 실시간으로 분석하고 대응할 수 있어 제조, 의료, 교통, 스마트홈 같은 분야에서 큰 가치를 만들 수 있습니다. 다만 개인정보, 편향, 설명 가능성, 책임 문제도 함께 발생하기 때문에 기술 도입과 함께 데이터 관리와 검증 체계가 반드시 필요하다고 생각합니다.

결국 좋은 AI 활용은 "AI를 쓴다"에서 끝나지 않는다. 어떤 데이터를 모을지, 어떤 문제를 해결할지, 어떤 기준으로 검증할지를 함께 설계해야 한다.
