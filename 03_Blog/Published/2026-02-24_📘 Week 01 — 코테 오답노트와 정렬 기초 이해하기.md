# 📘 Week 01 — 코테 오답노트와 정렬 기초 이해하기

## 1️⃣ 예상치 못한 음수 결과와 해결 방법

연산 결과가 음수로 나오거나 이상한 값이 나오는 경우는  
`int` 범위를 초과했기 때문일 가능성이 크다.

- `int` 범위: 약 ±21억
- 큰 수 연산 시 → `long long` 사용

```cpp
long long sum = 0;
```

큰 수가 나올 수 있는 문제는 처음부터 자료형을 확인하는 습관이 필요하다.

---

## 2️⃣ 시간 초과의 원인을 찾아 해결하기

시간 초과는 크게 두 가지 원인이 있다.

1. **로직이 느린 경우** → 알고리즘을 다시 생각해야 한다.
2. **입출력이 느린 경우**

입출력 최적화 방법:

### C 스타일 입출력 사용
```cpp
scanf();
printf();
```

### C++에서 속도 향상
```cpp
ios::sync_with_stdio(false);
cin.tie(NULL);
```

입출력만 바꿔도 실행 시간이 크게 줄어들 수 있다.

---

## 3️⃣ 인덱스에 의미 부여하여 풀어 보기

값을 정렬하는 대신,  
**배열의 인덱스를 활용해 정렬 효과를 내는 방법**이다.

1. 배열 선언
2. 입력 받은 숫자 위치를 1로 표시
3. 배열을 처음부터 탐색하며 1인 인덱스를 출력

```cpp
int arr[1001] = {0};
arr[number] = 1;
```

이후 `arr[i] == 1` 인 인덱스를 출력하면 자동으로 오름차순 정렬이 된다.

---

## 4️⃣ 나머지 연산의 중요성 알아보기

나머지 연산은 **분배법칙이 적용된다.**

✔ 덧셈  
`(A + B) % C = (A % C + B % C) % C`

✔ 뺄셈  
`(A - B) % C = (A % C - B % C) % C`

✔ 곱셈  
`(A × B) % C = (A % C × B % C) % C`

❌ 나눗셈은 바로 적용되지 않는다.

예제: 1부터 50까지 곱한 뒤 10007로 나머지 구하기

```cpp
long long result = 1;
for(int i = 1; i <= 50; i++) {
    result = (result * i) % 10007;
}
```

중간마다 나머지를 구하면 오버플로우를 방지할 수 있다.

# 🔽 정렬 기초 다지기

## 5️⃣ 기본 정렬 (sort 사용)

### 오름차순
```cpp
sort(arr, arr + N);
```

### 내림차순
```cpp
sort(arr, arr + N, greater<int>());
```

---

## 6️⃣ 다중 조건 정렬 익히기

### 1) 사용자 정의 비교 함수

영어 점수가 높은 순,  
영어가 같으면 수학 점수가 높은 순 정렬

```cpp
struct Score {
    int english;
    int math;
};

bool compare(Score a, Score b) {
    if(a.english == b.english)
        return a.math > b.math;
    return a.english > b.english;
}

sort(v.begin(), v.end(), compare);
```

---

### 2) 구조체 내부에서 연산자 오버로딩

```cpp
struct Score {
    int english;
    int math;

    bool operator<(const Score& other) const {
        if(english == other.english)
            return math > other.math;
        return english > other.english;
    }
};
```

이후 `sort(v.begin(), v.end());` 만 호출하면 된다.

## 7️⃣ 이차원 벡터 사용하기

그래프 표현 시 많이 사용된다.

```cpp
struct Edge {
    int endNode;
    int value;
};

vector<vector<Edge>> graph;
```

예시 입력:
```
3 4
1 2 3
2 1 10
1 3 7
3 2 6
```

의미:
- 노드 3개
- 간선 4개
- (시작노드, 도착노드, 가중치)

---

### 🔹 2차원 벡터 입력 후 1번 노드 출력 예시 코드

```cpp
#include <iostream>
#include <vector>
using namespace std;

struct Edge {
    int endNode;
    int value;
};

int main() {

    int node, edge;
    cin >> node >> edge;

    vector<vector<Edge>> graph(node + 1);

    for(int i = 0; i < edge; i++) {
        int start, end, value;
        cin >> start >> end >> value;

        graph[start].push_back({end, value});
    }

    // 1번 노드와 연결된 간선 출력
    for(int i = 0; i < graph[1].size(); i++) {
        cout << "도착 노드: " << graph[1][i].endNode
             << ", 가중치: " << graph[1][i].value << "\n";
    }

    return 0;
}
```

# 📌 정리

- 큰 수는 `long long`
- 시간 초과는 로직 또는 입출력 문제
- 인덱스를 활용하면 정렬을 빠르게 구현 가능
- 나머지 연산은 분배법칙 기억하기
- `sort`와 비교 함수는 코테 필수
- 그래프는 2차원 벡터로 표현
