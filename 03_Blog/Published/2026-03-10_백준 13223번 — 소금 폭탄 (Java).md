# 백준 13223번 — 소금 폭탄 (Java)

## 문제 설명

철수는 자동 로봇팔을 이용해 특정 시간이 되었을 때 물컵에 소금을 넣도록 프로그램을 작성하려고 한다.
현재 시각과 소금을 투하할 시각이 주어졌을 때, **현재 시각부터 소금을 투하할 시각까지 남은 시간**을 계산하는 문제이다.

단, 다음 조건을 만족해야 한다.

* 현재 시간과 목표 시간은 `hh:mm:ss` 형식으로 주어진다.
* 결과는 **1초 이상 24시간 이하**이다.
* 만약 목표 시간이 현재 시간보다 이전이라면 **다음 날의 시간**으로 계산해야 한다.

---

## 입력

첫째 줄: 현재 시각

```text
hh:mm:ss
```

둘째 줄: 소금을 투하할 시각

```text
hh:mm:ss
```

시간 범위

```text
0 ≤ h ≤ 23
0 ≤ m ≤ 59
0 ≤ s ≤ 59
```

---

## 출력

현재 시각부터 목표 시각까지 **남은 시간**을 `hh:mm:ss` 형식으로 출력한다.

---

## 입력 예시

```text
20:00:00
04:00:00
```

## 출력 예시

```text
08:00:00
```

---

# 문제 해결 아이디어

이 문제의 핵심은 **두 시간의 차이를 계산하는 것**이다.

하지만 단순히

```text
목표 시간 - 현재 시간
```

을 하면 다음과 같은 문제가 생길 수 있다.

예

```text
현재 시간 : 20:00:00
목표 시간 : 04:00:00
```

이 경우 목표 시간이 더 작기 때문에 **다음 날로 넘어간 상황**이다.

따라서 이런 경우에는

```text
24시간을 더해 계산
```

해야 한다.

---

# 내가 작성한 풀이

처음에는 시, 분, 초를 각각 비교하면서 **직접 빌림(borrow)** 처리를 하는 방식으로 구현했다.

예를 들어

* 초가 부족하면 분에서 1을 빌리고
* 분이 부족하면 시에서 1을 빌리는 방식이다.

## 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        String[] arr = sc.nextLine().split(":");

        int[] input = new int[arr.length];

        for(int i = 0; i < arr.length; i++){
            input[i] = Integer.parseInt(arr[i]);
        }

        String[] arr2 = sc.nextLine().split(":");

        int[] input2 = new int[arr.length];

        for(int i = 0; i < arr2.length; i++){
            input2[i] = Integer.parseInt(arr2[i]);
        }
        
        int time[] = new int[3];
        
        for(int i = 0; i < 3; i++) {

            if(input[i] > input2[i] && i == 0) {
                time[i] = input2[i] - input[i] + 24;
            }

            else if(input[i] > input2[i]) {

                time[i] = input2[i] - input[i] + 60;

                if(time[i - 1] == 0 && i == 1) {
                    time[i-1] = 23;
                }

                else if(time[i - 1] == 0 && i == 2) {
                    time[i-1] = 59;

                    if(time[i-2] == 0)
                        time[i-2] = 23;
                    else
                        time[i-2]--;
                }

                else {
                    time[i -1]--;
                }
            }

            else {
                time[i] = input2[i] - input[i];
            }
        }

        if(time[0] == 0 && time[1] == 0 && time[2] == 0)
            System.out.print("24:00:00");
        else
            System.out.printf("%02d:%02d:%02d",time[0],time[1],time[2]);
    }
}
```

이 방식은 사람이 직접 시계 계산을 하는 방식과 비슷하지만,
**빌림 처리와 예외 상황이 많아 로직이 복잡해지는 단점**이 있다.

---

# 강의 풀이 방식

강의에서는 시간을 **모두 초 단위로 변환**한 뒤 계산하는 방법을 사용했다.

## 풀이 과정

1. 현재 시간을 **초로 변환**
2. 목표 시간을 **초로 변환**
3. 두 시간의 차이를 계산
4. 음수가 나오면 **24시간을 더해 보정**
5. 다시 시, 분, 초로 변환

## 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        String[] start = sc.nextLine().split(":");
        String[] end = sc.nextLine().split(":");

        int startHour = Integer.parseInt(start[0]);
        int startMinute = Integer.parseInt(start[1]);
        int startSecond = Integer.parseInt(start[2]);

        int startTotalSecond = startHour * 3600 + startMinute * 60 + startSecond;

        int endHour = Integer.parseInt(end[0]);
        int endMinute = Integer.parseInt(end[1]);
        int endSecond = Integer.parseInt(end[2]);

        int endTotalSecond = endHour * 3600 + endMinute * 60 + endSecond;

        int newTotalSecond = endTotalSecond - startTotalSecond;

        if (newTotalSecond <= 0) {
            newTotalSecond += 24 * 3600;
        }

        int newHour = newTotalSecond / 3600;
        int newMinute = (newTotalSecond % 3600) / 60;
        int newSecond = newTotalSecond % 60;

        String ans = String.format("%02d:%02d:%02d", newHour, newMinute, newSecond);

        System.out.println(ans);
    }
}
```

---

# 두 풀이 방식 비교

| 방식           | 특징                 |
| ------------ | ------------------ |
| 시·분·초 직접 계산  | 빌림 처리가 필요하고 로직이 복잡 |
| 초 단위 변환 후 계산 | 코드가 간단하고 예외 처리가 쉬움 |

시간 계산 문제에서는 보통

```text
시간 → 초 변환 → 계산 → 다시 시간 변환
```

방식을 사용하는 것이 가장 안정적이다.

---

# 핵심 개념

이 문제에서 중요한 개념은 다음과 같다.

1. 시간을 **초 단위로 변환하는 방법**

```text
총 초 = 시 × 3600 + 분 × 60 + 초
```

2. 다음 날로 넘어가는 경우 처리

```text
차이 ≤ 0 이면 24시간(86400초) 더하기
```

3. 다시 시·분·초로 변환

```text
시 = 초 / 3600
분 = (초 % 3600) / 60
초 = 초 % 60
```

---

# 정리

처음에는 시, 분, 초를 각각 비교하며 직접 계산하는 방식으로 접근했지만
빌림 처리와 예외 상황 때문에 코드가 복잡해졌다.

강의에서는 시간을 모두 **초 단위로 변환한 뒤 차이를 계산하는 방식**을 사용했는데,
이 방법이 훨씬 간단하고 실수 가능성도 적다는 것을 알게 되었다.

시간 계산 문제에서는

```text
시간을 하나의 단위(초)로 변환하여 계산하는 방법
```

이 가장 정석적인 접근 방식이라고 느꼈다.
