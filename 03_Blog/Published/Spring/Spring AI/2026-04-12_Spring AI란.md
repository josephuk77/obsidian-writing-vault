# Spring AI란?

# Spring AI란? 스터디 전에 조사하면서 정리해봤다

요즘 개발하면서 AI를 서비스에 붙이는 게 점점 중요해지고 있다  

그래서 이번 스터디 주제로 Spring AI를 다루게 됐고  
들어가기 전에 개념부터 정리해봤다  

---

# Spring AI란?

Spring AI는  

👉 **Spring에서 AI 기능을 쉽게 사용할 수 있도록 만든 프레임워크**

라고 보면 된다  

기존 Spring 구조(Controller, Service 등)를 유지하면서  
LLM 기능을 자연스럽게 추가할 수 있도록 설계된 것이 특징이다  

---

# 기존 방식과 뭐가 다른가

Spring에서 AI 기능을 붙이는 방법은 크게 두 가지였다  

## 1) Python을 따로 사용하는 방식

- Python에서 AI 처리  
- Spring에서 Python 서버 호출  
- 결과를 다시 받아서 사용  

이 구조의 문제는

- 서버가 나뉘어서 관리가 복잡해지고  
- 배포도 따로 해야 하고  
- 유지보수 난이도가 올라간다는 점이다  

---

## 2) Spring에서 API 직접 호출

이 방법도 많이 사용된다  

하지만 실제로 구현해보면

- API 엔드포인트 직접 관리  
- 요청 JSON 직접 구성  
- 응답 JSON 직접 파싱  

이런 작업을 계속 반복해야 한다  

👉 결국 AI 로직보다 부수적인 코드가 많아진다  

---

# 그래서 Spring AI가 나온 이유

Spring AI는 위 문제를 해결하기 위해 만들어졌다  

핵심은 간단하다  

👉 **AI 호출을 Spring 방식으로 추상화했다는 점**

즉

- HTTP 요청을 직접 만들 필요 없고  
- JSON을 직접 파싱할 필요도 없고  
- 동일한 방식으로 여러 모델을 사용할 수 있다  

---

# 코드로 간단 비교

같은 기능 기준이다  

👉 "자기소개 한 줄 피드백"

---

## Python
```python
from openai import OpenAI

client = OpenAI(api_key="YOUR_API_KEY")

def feedback(intro):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "너는 자기소개 피드백 코치야"},
            {"role": "user", "content": intro}
        ]
    )

    return response.choices[0].message.content
```
빠르게 구현할 수 있지만  
Spring과 분리된 구조가 된다  

---

## Spring 직접 호출
```java
Map response = restClient.post()
        .uri("/chat/completions")
        .body(body)
        .retrieve()
        .body(Map.class);
```
가능은 하지만  

- 요청 구성  
- 응답 파싱  
- 모델 마다 다른 응답

이런 코드가 계속 필요하다  

---

## Spring AI
```java
return chatClient.prompt()
        .system("너는 자기소개 피드백 코치야")
        .user(intro)
        .call()
        .content();
```
👉 핵심 로직만 남는다  

이게 가장 큰 차이다  

---

# 정리해보면

Spring AI의 핵심은 세 가지라고 생각했다  

### 1) 코드 단순화

복잡한 HTTP, JSON 처리를 줄여준다  

---

### 2) 구조 일관성

Spring 구조 안에서 AI 기능을 그대로 사용할 수 있다  

---

### 3) 확장성

모델이 바뀌어도 동일한 방식으로 사용할 수 있다  

---

# 언제 쓰면 좋을까

## 👍 적합한 경우

- Spring 기반 백엔드에서 AI 기능을 추가할 때  
- 챗봇, 요약, 추천 같은 기능 구현할 때  
- 서비스 기능 안에 AI를 자연스럽게 넣고 싶을 때  

---

## 👎 적합하지 않은 경우

- AI 자체가 핵심인 서비스  
- 모델 학습, 파인튜닝이 필요한 경우  
- Python 기반 AI 생태계를 적극 활용해야 하는 경우  

---

# 한 줄 정리

👉 Spring AI는  
Spring 애플리케이션 안에서 AI를 더 쉽게 사용하게 해주는 프레임워크다
