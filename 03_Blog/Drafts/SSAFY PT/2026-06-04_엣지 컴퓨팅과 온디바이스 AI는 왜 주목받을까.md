# 엣지 컴퓨팅과 온디바이스 AI는 왜 주목받을까

## 개념 한 줄 정리

| 개념 | 간단한 설명 |
| --- | --- |
| 엣지 컴퓨팅 | 데이터를 멀리 있는 클라우드로 보내기 전에 현장 가까이에서 처리하는 방식 |
| 온디바이스 AI | 스마트폰, 카메라, 차량 같은 기기 내부에서 AI 추론을 수행하는 방식 |
| NPU | AI 연산에 특화된 반도체 |
| 모델 경량화 | 기기에서 실행할 수 있도록 AI 모델을 작게 만드는 기술 |
| OTA 업데이트 | 원격으로 기기 소프트웨어나 모델을 업데이트하는 방식 |
| 로컬 큐 | 네트워크 장애 시 데이터를 기기나 현장 시스템에 임시 저장하는 구조 |

## 글을 쓰는 이유

AI와 IoT가 확산될수록 모든 데이터를 클라우드로 보내는 방식에는 한계가 생긴다.

자율주행, 스마트팩토리, 헬스케어, CCTV 관제처럼 실시간 판단이 중요한 서비스는 응답 지연이 치명적일 수 있다. 개인정보가 포함된 데이터를 외부 서버로 보내는 것도 부담이다.

그래서 데이터가 발생하는 현장 가까이에서 처리하는 엣지 컴퓨팅과, 기기 내부에서 AI를 실행하는 온디바이스 AI가 주목받고 있다.

## 먼저 결론부터

엣지 컴퓨팅과 온디바이스 AI는 클라우드의 대체가 아니라 보완이다.

클라우드는 대규모 저장, 학습, 통합 분석에 강하다. 반면 엣지는 실시간 처리, 개인정보 보호, 네트워크 장애 대응에 강하다.

좋은 구조는 모든 것을 클라우드로 보내는 것이 아니라, 현장에서 처리할 것과 클라우드에서 처리할 것을 나누는 것이다.

## 개념

일반적인 클라우드 기반 AI는 데이터를 서버로 보내고, 서버에서 분석한 뒤 결과를 돌려준다.

```text
기기
-> 클라우드 서버
-> AI 분석
-> 결과 반환
```

엣지 컴퓨팅은 일부 처리를 현장 가까이에서 수행한다.

```text
기기
-> 엣지 장비에서 1차 처리
-> 필요한 데이터만 클라우드 전송
-> 통합 분석과 저장
```

온디바이스 AI는 스마트폰, 카메라, 자동차, 웨어러블 기기 내부에서 AI 추론을 수행한다. 예를 들어 스마트폰의 음성 인식, 카메라의 사람 감지, 차량의 실시간 판단 등이 있다.

## 등장 배경

- IoT 기기가 늘면서 클라우드로 보내야 할 데이터가 많아졌다.
- 자율주행, 스마트팩토리, 헬스케어처럼 지연 시간이 중요한 서비스가 늘었다.
- 개인정보를 외부 서버로 보내지 않고 처리해야 하는 요구가 커졌다.
- AI 반도체와 모바일 NPU 성능이 좋아졌다.
- 네트워크가 불안정한 환경에서도 서비스가 동작해야 한다.

## 장점

- 응답 속도가 빠르다.
- 네트워크가 불안정해도 일부 기능을 유지할 수 있다.
- 개인정보 보호에 유리하다.
- 클라우드 전송 비용을 줄일 수 있다.
- 실시간 제어가 필요한 서비스에 적합하다.

## 단점/리스크

- 기기 성능과 배터리 한계가 있다.
- 모델 업데이트가 어렵다.
- 기기별 성능 차이로 품질 관리가 어렵다.
- 현장 기기가 해킹되면 위험할 수 있다.
- 클라우드와 엣지 사이의 역할 분리가 어려울 수 있다.

## 기술적 해결방안

| 리스크 | 기술적 해결방안 |
| --- | --- |
| 기기 성능 부족 | 모델 경량화, 양자화, 캐싱, NPU 활용 |
| 배터리 문제 | 저전력 모드, 이벤트 기반 추론, 추론 주기 조절 |
| 모델 업데이트 문제 | OTA 업데이트, 버전 관리, 단계적 배포, 롤백 |
| 네트워크 장애 | 로컬 큐, 재전송, 오프라인 모드 |
| 보안 문제 | Secure Boot, 기기 인증, 암호화, 원격 무결성 검증 |
| 품질 차이 | 기기별 테스트, 원격 모니터링, A/B 테스트 |

예를 들어 CCTV에서 모든 영상을 클라우드로 보내면 비용과 개인정보 부담이 크다. 카메라 가까이에서 사람이나 이상 행동을 먼저 감지하고, 필요한 이벤트만 서버로 보내면 부담을 줄일 수 있다.

## 활용사례

- 스마트폰 음성 인식과 번역
- CCTV 이상 행동 감지
- 자율주행 차량의 실시간 판단
- 스마트팩토리 설비 이상 감지
- 웨어러블 건강 데이터 분석
- 스마트홈 기기 자동 제어

## 예상 질문과 답변

### 엣지 컴퓨팅과 클라우드의 차이는 무엇인가요?

클라우드는 중앙 서버에서 대규모 저장과 분석을 수행하고, 엣지 컴퓨팅은 데이터가 발생하는 현장 가까이에서 빠르게 처리합니다.

### 온디바이스 AI의 장점은 무엇인가요?

응답 속도가 빠르고 개인정보 보호에 유리합니다. 네트워크가 불안정해도 일부 기능을 유지할 수 있습니다.

### 모델 업데이트 문제는 어떻게 해결할 수 있나요?

OTA 업데이트를 적용하고, 버전 관리를 하며, 문제가 생기면 이전 모델로 롤백할 수 있는 구조를 만들어야 합니다.

## 정리

엣지 컴퓨팅과 온디바이스 AI는 클라우드 중심 구조의 한계를 보완한다.

면접에서 사용할 결론:

> 엣지 컴퓨팅과 온디바이스 AI는 데이터를 현장 가까이에서 처리해 지연 시간, 개인정보, 네트워크 의존성 문제를 줄이는 기술입니다. 개발자는 모델 경량화, OTA 업데이트, 로컬 큐, 기기 인증 같은 구조를 함께 고려해야 안정적인 서비스를 만들 수 있습니다.
